General

Problem Solving

Problem solving is often more than half of a developer's tasks. Personally whenever I run into a problem I try and look for a use case example of the exact thing I'm trying to achieve. If this doesn't exist I will move to documentation before heading over to forums and requesting help. If the problem is contained within a bug I will try and resolve it with developer tools and code validators first to check for any syntax errors before moving onto forums.

Code validation

All my work is validated using the following:

  • - Sites are tested in Chrome for Windows, and FireFox for Windows
  • - HTML validated via the W3C Markup Validation Service
  • - CSS validated via the W3C CSS Validation Service
  • - JS checked via JS Hint and JS console

desgin

I aim to create all of my projects in a mobile-first responsive design pattern. This means that I will create a fully funtioning project on the smallest mobile size screen first and then add in media-queries to adjust for larger screen sizes.

Version Control

I use GitHub as my preferred version control platform.
